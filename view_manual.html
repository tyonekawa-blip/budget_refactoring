dashboard ダッシュボード と 詳細分析

ダッシュボード： 選択した期間（単月や四半期）、または年度累計（YTD）でのチーム全体の予実状況、進捗率、前年比をサマリ表示します。コスト構成比のグラフも確認できます。

詳細分析： 勘定科目 ＞ チーム ＞ セグメント ＞ 取引先 の4階層でドリルダウン（掘り下げ）ができる分析画面です。差異の要因を特定し、末端の明細（セグメント・取引先）に対して「入力」ボタンからポップアップで要因コメントを保存することができます。

assessment 報告資料 (サマリ・抜粋)

報告資料 (サマリ)： 経営層やマネージャー向けに、各チームの成績を「天気アイコン」と合わせて直感的に表示します。コメント欄はなく、数値と前年比で全体感を把握するためのビューです。

報告資料 (抜粋)： 「詳細分析」画面でコメントが入力された明細だけを自動で抽出し、報告書フォーマットにまとめて表示します。

emoji_events 創会用レポート ＆ 財務戦略会議用

創会用レポート： 計画精度の高いチームを称える「ランキング」や「ピッタリ賞」を表示します。画面いっぱいのレイアウトで取引先ごとのコメントが確認でき、「チームをPDF出力」ボタンからA4横1枚の報告書を印刷できます。天気アイコンの基準（％または金額）もここで設定可能です。

財務戦略会議用： システム費や外注費など、事前に設定した「重要投資科目」の消化状況と、予算未消化の取引先（黄色アラート）を経営陣に報告するための専用レポートです。

manage_accounts 管理者向け機能一覧

  • 取込管理： 指定したGoogle Driveのフォルダから、CSV形式の「予算データ」および「実績データ（速報・確定）」をGAS経由で一括インポートします。
  • 実績データ 表示設定： 月ごとに、ダッシュボード等で「速報値」を見るか「確定値」を見るかを切り替えます。
  • 勘定科目 PL表示設定： 基幹システムのマスタから取り込んだ科目のうち、このシステム（PL）で表示するものにチェックを入れて保存します。
  • チーム編成設定： 未割り当てのセグメントをドラッグ＆ドロップして、報告用の「チーム（部署）」を作ります。
  • ユーザー権限設定： このシステムにアクセスできるユーザーの追加・権限（Admin/User）・閲覧可能メニューを設定します。
  • 初期年度・締め処理： システム全体の基準年度を変更したり、月次を「ロック（確定）」してコメントの改ざんを防ぎます。

folder_open システムファイル・スクリプト構成

本システムはGoogle Apps Script (GAS)をバックエンド、HTML/JavaScriptをフロントエンドとしたSPA（Single Page Application）構成です。

📄 バックエンド (GAS: .gsファイル)

ファイル名役割・処理内容
main.gsシステム全体のエントリーポイント（doGetによる画面描画）。エラーやアクションのログをシートに書き込む機能も担う。
api_data.gsスプレッドシートからマスタデータ、予算・実績キャッシュデータ、コメントを抽出し、フロントエンドへJSON形式（globalPackage）で渡す。
api_import.gsDriveからCSVファイルを検索し、マスタと突合しながら実績・予算データを「DATA_CACHE_FYxx」シートに展開・上書きするバッチ処理。
api_settings.gsユーザー、チーム、天気基準、財務レポート設定、コメント等の保存処理。スプレッドシートの各種保管シートへ直接書き込みを行う。

🌐 フロントエンド (HTML/JSファイル)

ファイル名役割・処理内容
index.html / css.html画面の大枠（サイドバー等）と、システム全体で共通利用されるCSSスタイル定義。
js_controller.html画面読み込み時の初期化処理、GASからのデータ受け取り、各画面の描画関数へのデータ分配を行う司令塔。
js_common.html金額のカンマ区切り（formatMoney）、パーセント計算、通信エラー処理など、全画面で使い回す共通関数群。
view_xxx / js_xxx (各画面) _dashboard: ダッシュボードのUIとグラフ描画
_drilldown: 詳細分析のドリルダウン（アコーディオン）とコメント入力ポップアップ
_report: 報告資料（サマリ/抜粋）のUIと天気アイコン描画
_report_zaimu: 財務戦略用レポートとCSV設定連携機能
_report_soukai: 創会用レポートと専用PDF横向き印刷機能
_settings系: 各種設定画面のUIと保存アクション
